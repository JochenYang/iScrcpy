# iScrcpy 사용자 가이드

## 빠른 시작

### 시스템 요구 사항

- Windows 10/11
- Android 5.0+ 기기
- USB 디버깅 활성화
- WiFi 네트워크 연결

### 첫 사용

1. iScrcpy를 다운로드하여 설치
2. Android 기기를 컴퓨터에 연결
3. 기기에서 USB 디버깅 승인
4. 기기 카드에서 "미러링 시작" 클릭

---

## 기기 연결

### USB 연결

1. USB 케이블로 스마트폰과 컴퓨터 연결
2. 승인 프롬프트에서 "허용" 탭
3. 기기가 USB 기기 목록에 표시됨
4. "미러링 시작" 클릭

### WiFi 연결

**방법 1: TCP/IP 모드 사용**

1. 먼저 USB로 연결
2. 기기 카드에서 "WiFi 활성화" 클릭
3. 기기 IP 얻기 완료 대기
4. USB 분리하고 WiFi 연결

**방법 2: 수동 연결**

1. 스마트폰과 컴퓨터가 같은 네트워크에 있는지 확인
2. 스마트폰에서 개발자 옵션 → 네트워크 디버깅 활성화
3. "WiFi 기기 추가" 클릭
4. 기기 IP 주소 입력 (예: `192.168.1.100:5555`)

## 표시 설정

"표시" 탭에서 설정:

### 비디오 설정

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| 최대 해상도 | 최대 비디오 크기 제한 | 1080p |
| 비디오 비트레이트 | 비디오 인코딩 품질 (Mbps) | 8 Mbps |
| 프레임 레이트 | 최대 프레임 레이트 제한 | 60 fps |
| 비디오 활성화 | 비디오 스트림 토글 | 켜짐 |
| 오디오 활성화 | 오디오 스트림 토글 | 켜짐 |

### 창 설정

| 옵션 | 설명 |
|------|------|
| 항상 위쪽 | 미러링 창을 위쪽에 유지 |
| 전체 화면 | 전체 화면 모드로 시작 |
| 유지각성 유지 | 미러링 중 기기 화면 켜짐 유지 |
| 테두리 없음 모드 | 창 제목 표시줄과 테두리 숨김 |
| 화면 보호기 비활성화 | 시스템 슬립 및 화면 보호기 방지 |

## 녹화

### 기본 녹화

1. 기기 카드에서 빨간 녹화 버튼 클릭
2. 다시 클릭하여 녹화 중지
3. 비디오는 기본적으로 다운로드 폴더에 저장

### 자동 녹화

"표시" → "녹화 설정"에서:

| 옵션 | 설명 |
|------|------|
| 자동 녹화 | 연결 시 자동으로 녹화 시작 |
| 오디오 녹화 | 비디오와 함께 오디오 녹화 |
| 저장 경로 | 사용자 정의 저장 위치 |
| 시간 제한 | 녹화 지속 시간 제한 (0 = 무제한) |

### 녹화 형식

- MP4, MKV, WEBM 형식 지원
- 비디오 코덱: H.264 / H.265 / AV1
- 오디오 코덱: Opus / AAC

## 카메라 미러링

### 카메라 미러링 활성화

1. "표시" → "카메라 설정"에서 활성화
2. 카메라 해상도와 프레임 레이트 선택
3. 기기 카드에서 카메라 버튼 클릭

### 카메라 설정

| 옵션 | 설명 |
|------|------|
| 카메라 해상도 | 640x480 ~ 3840x2160 |
| 카메라 프레임 레이트 | 15 ~ 120 fps |

## 인코딩 설정

"인코딩" 탭에서 고급 인코딩 옵션:

### 비디오 코덱

| 코덱 | 설명 |
|------|------|
| H.264 | 기본값, 최적의 호환성 |
| H.265 | 높은 압축률, 기기 지원 필요 |
| AV1 | 최신 코덱, 기기 지원 필요 |

### 오디오 코덱

| 코덱 | 설명 |
|------|------|
| Opus | 기본값, 권장 |
| AAC | 더 나은 호환성 |

### 비트레이트 모드

| 모드 | 설명 |
|------|------|
| VBR | 가변 비트레이트, 더 작은 파일 |
| CBR | 고정 비트레이트, 더 나은 안정성 |

## 서버 설정

"서버" 탭으로 이동:

### 터널 모드

- **역방향 터널 (기본값)**: `adb reverse`로 연결
- **정방향 터널**: `adb forward`로 연결

### 자동 정리

활성화 시 연결 해제 시 scrcpy 서버 프로세스를 자동으로 정리.

## FAQ

### Q: 기기가 반응하지 않음?

1. USB 케이블이 데이터 전송을 지원하는지 확인
2. 기기에서 USB 디버깅이 활성화되어 있는지 확인
3. USB 케이블을 다시 연결해 봄
4. 올바른 ADB 드라이버가 설치되어 있는지 확인

### Q: 미러링이 끊김?

1. 최대 해상도 낮추기
2. 비디오 비트레이트 줄이기
3. 프레임 레이트 제한 낮추기
4. WiFi 5GHz 네트워크 사용 시도

### Q: 소리가 없음?

1. "표시" 설정에서 오디오가 활성화되어 있는지 확인
2. 컴퓨터音量 설정 확인
3. 일부 기기는 오디오 전송을 지원하지 않을 수 있음

### Q: 녹화 파일이 손상됨?

1. "녹화 중지" 버튼으로 정상적으로 종료
2. 녹화 중 강제 연결 해제 피하기
3. iScrcpy는 손상된 파일을 자동으로 복구하려 함

### Q: 전체 화면을 어떻게 종료?

`MOD+f` 단축키를 누르거나 전체 화면 옵션 없이 미러링 재시작.

## 단축키

| 단축키 | 기능 |
|--------|------|
| `MOD+f` | 전체 화면 토글 |
| `MOD+좌/우` | 화면 회전 |
| `MOD+g` | 검은 화면 토글 |
| `MOD+r` | 기기 회전 토글 |
| `Ctrl+클릭+드래그` | 우클릭 시뮬레이션 |
| `휠` | 볼륨 버튼 시뮬레이션 |
