# iScrcpy ユーザーガイド

iScrcpy は [scrcpy](https://github.com/Genymobile/scrcpy) をベースとした Android デバイスミラーリングツールで、シンプルな GUI と豊富なカスタマイズ機能を提供します。

## クイックスタート

### システム要件

- Windows 10/11
- Android 5.0+ デバイス
- USB デバッグが有効
- WiFi ネットワーク接続

### 初回使用

1. iScrcpy をダウンロードしてインストール
2. Android デバイスをパソコンに接続
3. デバイスで USB デバッグを許可
4. デバイスカードの「ミラーリング開始」ボタンをクリック

---

## デバイスを接続する

### USB 接続

1. USB ケーブルでスマホとパソコンを接続
2. 手机で許可プロンプトが表示されたら、「許可」をクリック
3. デバイスが USB デバイスリストに表示されます
4. 「ミラリング開始」をクリック

### WiFi 接続

**方法 1: TCP/IP モードを使用**

1. まず USB でデバイスを接続
2. デバイスカードの「WiFi 有効化」ボタンをクリック
3. デバイス IP 取得完了を待つ
4. USB を外して WiFi で接続

**方法 2: 手動接続**

1. スマホとパソコンが同じネットワークに接続されていることを確認
2. スマホで開発者オプション → ネットワークデバッグを有効化
3. 「WiFi デバイスを追加」をクリック
4. デバイス IP アドレスを入力（例: `192.168.1.100:5555`）

## 表示設定

「表示」タブで設定：

### ビデオ設定

| オプション | 説明 | デフォルト |
|-----------|------|-----------|
| 最大解像度 | 最大ビデオサイズを制限 | 1080p |
| ビデオビットレート | ビデオエンコーディング品質 (Mbps) | 8 Mbps |
| フレームレート | 最大フレームレート制限 | 60 fps |
| ビデオを有効化 | ビデオストリームを切り替え | オン |
| オーディオを有効化 | オーディオストリームを切り替え | オン |

### ウィンドウ設定

| オプション | 説明 |
|-----------|------|
| 最前面に固定 | ミラリングウィンドウを最前面に保持 |
| 全画面モード | 全画面で起動 |
| スリープ防止 | ミラーリング中にデバイス画面をオンのまま |
| ボーダーレスモード | ウィンドウタイトルバーと枠を非表示 |
| スクリーンセーバー無効化 | システムのスリープとスクリーンセーバーを防止 |

## 録画

### 基本的な録画

1. デバイスカードの赤い録画ボタンをクリック
2. もう一度クリックして録画を停止
3. ビデオはデフォルトでダウンロードフォルダに保存

### 自動録画

「表示」→「録画設定」で：

| オプション | 説明 |
|-----------|------|
| 自動録画 | 接続時に自動的に録画開始 |
| 音声を録音 | ビデオと一緒に音声も録音 |
| 保存パス | 保存場所を選択 |
| 時間制限 | 録画時間の制限（0 = 無制限） |

### 録画形式

- MP4、MKV、WEBM 形式をサポート
- ビデオコーデック: H.264 / H.265 / AV1
- オーディオコーデック: Opus / AAC

## カメラミラーリング

### カメラミラーリングを有効化

1. 「表示」→「カメラ設定」で有効化
2. カメラ解像度とフレームレートを選択
3. デバイスカードのカメラボタンをクリック

### カメラ設定

| オプション | 説明 |
|-----------|------|
| カメラ解像度 | 640x480 ~ 3840x2160 |
| カメラフレームレート | 15 ~ 120 fps |

## エンコーディング設定

「エンコーディング」タブで詳細設定：

### ビデオコーデック

| コーデック | 説明 |
|-----------|------|
| H.264 | デフォルト、最適互換性 |
| H.265 | 更高的圧縮率、デバイスサポートが必要 |
| AV1 | 最新のコーデック、デバイスサポートが必要 |

### オーディオコーデック

| コーデック | 説明 |
|-----------|------|
| Opus | デフォルト、推奨 |
| AAC | 更好的兼容性 |

### ビットレートモード

| モード | 説明 |
|-------|------|
| VBR | 可変ビットレート、ファイルサイズ小 |
| CBR | 固定ビットレート、より安定した品質 |

## サーバー設定

「サーバー」タブ：

### トンネルモード

- **リバーストンネル (デフォルト)**: `adb reverse` で接続
- **フォワードトンネル**: `adb forward` で接続

### 自動クリーンアップ

有効にすると、接続解除時に scrcpy サーバープロセスを自動的にクリーンアップ。

## FAQ

### Q: デバイスに反応がない？

1. USB ケーブルが数据传输をサポートしているか確認
2. デバイスで USB デバッグが有効になっていることを確認
3. USB ケーブルを再度接続し直す
4. 正しい ADB ドライバーがインストールされているか確認

### Q: ミラリングが途切れる？

1. 最大解像度を下げる
2. ビデオビットレートを下げる
3. フレームレート制限を下げる
4. WiFi 5GHz ネットワークを試す

### Q: 音声が聞こえない？

1. 「表示」設定でオーディオが有効になっているか確認
2. 电脑の音量設定を確認
3. 一部のデバイスは音声伝送をサポートしていない場合があり

### Q: 録画ファイルが壊れた？

1. 「録画停止」按钮を，正常的に終了
2. 録画中に强制的に切断しない
3. iScrcpy は损坏したファイルを自動的に修復しようとします

### Q: 全画面を終了するには？

`MOD+f` ホットキーを押すか、全画面オプションなしでミラーリングを再起動。

## ホットキー

| ホットキー | 機能 |
|-----------|------|
| `MOD+f` | 全画面切り替え |
| `MOD+左/右` | 画面回転 |
| `MOD+g` | 黑画面切り替え |
| `MOD+r` | デバイス回転切り替え |
| `Ctrl+クリック+ドラッグ` | 右クリックをシミュレート |
| `スクロール wheel` | 音量按钮をシミュレート |
